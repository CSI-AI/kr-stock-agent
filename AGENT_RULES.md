# AGENT_RULES.md — 와바바 AI 개발 운영 규칙

> 이 파일은 Claude Code가 와바바 프로젝트에서 따라야 할 개발·운영 규칙을 고정합니다.
> 이후 모든 작업의 기준 문서입니다. 규칙 변경은 사용자 명시적 지시로만 가능합니다.

---

## 프로젝트 정보

- **프로젝트명**: 와바바 (Why Buy & Bye?)
- **목적**: 국내주식 가치성장 투자 운영 에이전트 웹앱
- **투자 방향**: 장기 가치투자 중심
- **성격**: 트레이딩 앱 아님. AI 추천 + 포트폴리오 운영 도구
- **운영 펀드**: 와바바펀드 (`portfolio.json`) · 와바바AI펀드 (`wababa-ai-portfolio.json`)

---

## 핵심 투자 철학

- 고성장 + 저평가 기업 우선 (ROE 높고 PER 낮은 종목)
- 성장 지속 가능성을 가장 중요하게 평가
- 뉴스·실적·산업 모멘텀을 수치 기반으로 함께 분석
- 단순 단타 매매 절대 금지
- 기준에 맞는 종목이 없으면 매수하지 않음 (억지 매수 금지)
- 현금 보유는 언제나 유효한 전략

---

## 추천 원칙

- 하루 최대 1~3개 종목 추천
- 근거 없는 추천 생성 금지
- 투자 이유는 구체적인 수치와 사실 기반으로 작성
- "실적 개선 기대", "성장 가능성" 같은 추상적 표현 최소화
- 실제 뉴스·재무 데이터·실적 기반 설명 우선
- 추천 근거는 `hypothesis` / `evidence` / `risk` 구조를 따름

---

## UI 원칙

- 초보 투자자도 이해할 수 있는 화면 구성
- 숫자 중심 표시: ROE / PER / PBR / 영업이익률 / 성장률 명확 노출
- 텍스트 과다 사용 금지 — 리포트는 메모처럼 짧고 직관적으로
- 가독성 최우선: 정보 밀도보다 이해 가능성 우선
- 두 펀드(와바바펀드 / 와바바AI펀드)는 화면에서 명확히 구분해 표시

---

## 개발 원칙

작업 순서를 반드시 지킵니다:

```
1. 조사 (관련 파일 읽기, 영향 범위 파악)
2. 계획 (변경 파일 목록·이유 사전 보고)
3. 승인 (사용자 확인 후 진행)
4. 수정 (최소 범위 수정)
5. 검증 (타입 체크 또는 빌드 확인)
```

- 무단 리팩토링 금지 — 기능 변경 없는 코드 정리도 사전 보고 필요
- 수정 범위는 항상 최소화
- 안정성 우선: 불확실하면 수정 전 사용자에게 보고
- 파일별 역할 분리 유지 (types / scoring / UI / data 경계 존중)

---

## 코드 구조 원칙

- `types/` — 타입 정의 전용. 로직 혼입 금지
- `lib/scoring/` — 점수 계산 로직 전용. UI 코드 혼입 금지
- `lib/strategy/` — 전략 엔진. UI 직접 의존 금지
- `app/` — UI와 API 라우트. 비즈니스 로직 직접 구현 금지
- `lib/mock/` — mock 데이터 전용. real 데이터 경로와 절대 혼합 금지
- UI 컴포넌트에 계산 로직 직접 작성 금지

---

## 데이터 원칙

- `recommendation-history.json` 구조(필드명·네임스페이스) 안정성 유지
  - 와바바펀드: `fundTradeResult` / `portfolioSummary`
  - 와바바AI펀드: `aiFundTradeResult` / `aiPortfolioSummary`
- `portfolio.json` / `wababa-ai-portfolio.json` 직접 수정 금지 (Python이 관리)
- 날짜 포맷은 `YYYY-MM-DD` 형식으로 일관성 유지
- `financial-universe-real.json`은 Python 산출물 — 직접 편집 금지
- 자동운용 로직(`wababa-auto-daily`, `build_recommendation_history.py`) 변경 시 반드시 사용자 보고 후 진행

## 위험 작업 금지 목록

| 금지 항목 | 이유 |
|-----------|------|
| `.env.local` 내용 출력 | API Key, 경로 등 민감 정보 포함 |
| API Key / Secret 출력 | 보안 |
| 대량 파일 삭제 | 복구 불가 |
| `git push` 무단 실행 | 원격 반영은 사용자가 직접 결정 |
| `git commit` 무단 실행 | 커밋 전 사용자 확인 필수 |
| `portfolio.json` 직접 수정 | 실제 펀드 상태 파괴 위험 |
| Python 스크립트 무단 실행 | 실제 거래 데이터 덮어쓰기 위험 |
| `autoTradingEnabled` 값 변경 | 자동 매매 ON/OFF에 직접 영향 |
